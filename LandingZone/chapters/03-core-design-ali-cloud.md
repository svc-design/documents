
# 个人版 Landing Zone 最小化清单（阿里云版）

## 1. 身份与访问控制（IAM 基线）
- **启用多因子认证 (MFA)**：保护主账号。
- **创建 RAM 用户**：不要直接用主账号日常操作。
  - 管理员用户：全局管理。
  - 运维/开发用户：日常工作，最小权限。
- **启用访问密钥管理**：为 CLI / SDK 生成 AK/SK，避免泄露。

## 2. 日志与审计
- 启用 **ActionTrail（操作审计）**。
- 设置日志投递到 **OSS + 日志服务 (SLS)**。
- 保留操作轨迹（谁在什么时候做了什么）。
- 开启 **OSS Bucket 日志**（如果存放重要数据）。

## 3. 安全与合规
- 开启 **阿里云 Config**：
  - 禁止公网 RDS。
  - ECS 必须绑定安全组。
  - 禁止 0.0.0.0:22 开放。
- 开通 **云安全中心（免费版即可）**：
  - 检测漏洞、暴露风险。
- **设置安全组基线**：
  - 默认拒绝，按需放行（22/80/443 等）。

## 4. 网络基线
- **创建 VPC 与子网**：
  - 生产子网：部署主要 ECS / RDS。
  - 测试子网：隔离实验环境。
- 禁用默认安全组全放行。
- 考虑 **NAT 网关 / 弹性公网 IP**：统一出口管理。

## 5. 资源与成本控制
- **标签管理 (Tagging)**：
  - 给每个资源加上 `Env=Dev/Prod`，`Owner=个人`。
- **预算与告警**：
  - 在费用中心设置预算。
  - 超出时短信/邮件提醒。

## 6. 自动化与 IaC
- 使用 **Terraform / ROS 最小化模板**：
  - 创建 VPC / 子网。
  - 创建 RAM 用户 + 基础策略。
  - 开启 ActionTrail。
  - 部署 ECS 并绑定安全组。
- 以后建新环境时，只需跑一遍 IaC 脚本即可复用。
